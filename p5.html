🎯 What is Data Cleaning and Preprocessing?

Before you can analyze data or build models, you need to clean and prepare it. Real-world data is messy! It has missing values, wrong entries, and inconsistencies.

Think of it this way:

🏗️ Building a house: You don't build on dirty ground. You first clean the foundation, remove rocks, level it, then build.

📊 Working with data: You don't analyze messy data. You first clean it, fix errors, remove duplicates, then analyze.

Data cleaning = 80% of data science work!

Topic 1: Handling Missing Values

🎯 What are Missing Values?

Missing values are empty or blank cells in your data where information should be but isn't.

Represented as:

  • ❌ NaN (Not a Number)
  • ❌ NULL
  • ❌ None
  • ❌ Empty cells

Real example:

A customer database where some customers didn't provide a phone number → Missing value!

❓ Why Do Missing Values Happen?

📋 User didn't enter data

Customer skipped optional form fields

⚠️ System error

Database crash lost some records

🔗 Data integration problem

Combining data from different sources, some fields don't match

📊 Not applicable

Field doesn't apply to that record (e.g., high school for college student)

🛠️ Strategies to Handle Missing Values

1. Delete (Remove) Missing Values ❌

Simply remove rows or columns with missing data

# Remove rows with ANY missing values
df_clean = df.dropna()
# Remove rows where 'Age' is missing
df_clean = df.dropna(subset=['Age'])
# Remove columns with missing values
df_clean = df.dropna(axis=1)

✅ Pros:

Simple, keeps only clean data

❌ Cons:

Loses data, may remove too much information

2. Fill with Mean, Median, or Mode 📊

Replace missing values with average or most common value

# Fill with mean (average)
df['Age'].fillna(df['Age'].mean())
# Fill with median
df['Salary'].fillna(df['Salary'].median())
# Fill with mode (most frequent)
df['Category'].fillna(df['Category'].mode()[0])

✅ Pros:

Keeps data, reasonable estimate

❌ Cons:

Assumes all missing values are similar (may not be accurate)

3. Forward/Backward Fill ⏩

Use value from previous or next row (good for time series data)

# Fill with previous value (forward fill)
df.fillna(method='ffill')
# Fill with next value (backward fill)
df.fillna(method='bfill')

✅ Pros:

Good for time-series data, preserves trends

❌ Cons:

Only works if data has temporal order

4. Interpolation 📈

Estimate missing values based on surrounding values

# Linear interpolation
df.interpolate(method='linear')

Example: Values are 10, __, __, 40

Interpolation fills as: 10, 20, 30, 40

✅ Pros:

Smart estimation based on pattern

❌ Cons:

Assumes linear relationship

🤔 Which Strategy to Use?

Missing value percentage:

  • • >50% missing → DELETE the column
  • • 5-50% missing → Fill with mean/median
  • • <5% missing → Delete rows or interpolate

Data type:

  • • Numeric (age, salary) → Mean/Median
  • • Time series → Forward/Backward fill
  • • Categorical (color, grade) → Mode

Topic 2: Handling Outliers

🎯 What are Outliers?

Outliers are data points that are extremely different from the rest. They're unusual values that stand out.

Real examples:

  • 📊 Student test scores: Most get 70-90, but one student got 5 (copied wrong answers?)
  • 💰 Employee salaries: Most earn 50,000-80,000, but CEO earns 5,000,000
  • 🛒 Customer purchases: Most buy $50 items, one customer buys $100,000 worth (bulk order?)

🔍 Types of Outliers

✓ Legitimate Outliers (Keep)

Real data that IS different - CEO's salary is legitimately high

Action: Keep it, it's real!

❌ Erroneous Outliers (Remove)

Data entry errors - temperature recorded as 999°C instead of 29.9°C

Action: Delete or fix it!

⚠️ Suspicious Outliers (Investigate)

Could be interesting or an error - unusual purchase behavior

Action: Investigate before deciding

🔎 How to Find Outliers

Use statistical methods (same as in Week 4!):

📊 IQR Method (Most Common)

Outlier = Value {'>'} (Q3 + 1.5×IQR) OR {'<'} (Q1 - 1.5×IQR)
import pandas as pd
import numpy as np
# Find outliers
Q1 = df['value'].quantile(0.25)
Q3 = df['value'].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers = df[(df['value'] < lower) |
(df['value'] > upper)]

📐 Z-Score Method

Outlier = |Z-Score| {'>'} 3
Z-Score = (Value - Mean) ÷ Std Dev
from scipy import stats
# Find outliers using Z-score
z_scores = np.abs(stats.zscore(df['value']))
outliers = df[z_scores > 3]

🛠️ How to Handle Outliers

1. Remove Outliers ❌

Delete rows with outlier values

# Remove outliers detected by IQR
df_clean = df[(df['value'] >= lower) &
(df['value'] <= upper)]

✅ Pros:

Clean data, won't affect analysis

❌ Cons:

Loses information, might remove important data

2. Cap or Floor (Winsorizing) 🎯

Replace extreme values with a maximum/minimum threshold

# Cap values to upper/lower bounds
df['value'] = df['value'].clip(lower=lower, upper=upper)
# Replace values > upper with upper
df.loc[df['value'] > upper, 'value'] = upper

Example: If salary cap is $100k, replace all $500k with $100k

✅ Pros:

Keeps data, reduces extreme values

❌ Cons:

Changes original values

3. Transform Data 🔄

Apply mathematical transformation to reduce impact of outliers

# Log transformation reduces extreme values
df['value_log'] = np.log(df['value'])
# Square root transformation
df['value_sqrt'] = np.sqrt(df['value'])

✅ Pros:

Keeps data, handles skewed data well

❌ Cons:

Harder to interpret results

4. Separate Analysis 📂

Keep outliers separate and analyze them differently

# Create two datasets
normal_data = df[(df['value'] >= lower) &
(df['value'] <= upper)]
outlier_data = df[(df['value'] < lower) |
(df['value'] > upper)]

✅ Pros:

Keep all data, understand outliers separately

❌ Cons:

Complex analysis with two datasets

Topic 3: Normalization and Transformation

🎯 What are Normalization and Transformation?

These techniques rescale or reshape your data to make it:

  • ✅ On the same scale (comparing 1-100 vs 1000-100000)
  • ✅ More suitable for algorithms
  • ✅ Easier to interpret

Real analogy:

🌍 Converting: 5 km vs 3000 meters → Both to meters: 5000m vs 3000m (now comparable!)

📏 Normalization (Scaling to 0-1)

Scale all values to a specific range, usually 0 to 1

Min-Max Normalization (Most Common)

Normalized = (Value - Min) ÷ (Max - Min)
from sklearn.preprocessing import MinMaxScaler
# Create scaler
scaler = MinMaxScaler()
# Apply to data
df_normalized = scaler.fit_transform(df[['Age']])

Example:

Ages: 20, 30, 40, 50 → Normalized: 0, 0.33, 0.67, 1.0

✅ When to use:

When you know the data range (e.g., test scores 0-100)

Z-Score Normalization (Standardization)

Z-Score = (Value - Mean) ÷ Standard Deviation
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
df_standardized = scaler.fit_transform(df[['Age']])

Example:

Mean=35, StdDev=7: Age 35 → 0, Age 42 → 1.0, Age 28 → -1.0

✅ When to use:

When data is normally distributed (like test scores)

🔄 Data Transformation

Change the shape or distribution of data to make it more suitable for analysis

📊 Log Transformation

Apply logarithm to compress large values, good for skewed data

import numpy as np
import pandas as pd
# Log transformation
df['income_log'] = np.log(df['income'])

Example:

Income: 1000, 10000, 100000 → Log: 3.0, 4.0, 5.0 (linear!)

✅ Best for:

Exponential/skewed data (income, prices, web traffic)

⚡ Power Transformations

Square root, cube root, or other power functions

# Square root
df['age_sqrt'] = np.sqrt(df['age'])
# Cube root
df['age_cbrt'] = np.cbrt(df['age'])

✅ Best for:

Moderately skewed data (less extreme than log)

🎁 Box-Cox Transformation

Automatic transformation that finds best power

from scipy.stats import boxcox
# Automatic transformation
transformed, lambda_param = boxcox(df['value'])
df['value_transformed'] = transformed

✅ Best for:

When you don't know which transformation to use

📊 Normalization vs Transformation

Aspect Normalization Transformation
Purpose Scale to 0-1 or mean=0 Change distribution shape
Output Range 0-1 or -3 to +3 Variable
Use Case Same scale comparison Fix skewed data
Examples Min-Max, Z-Score Log, Square Root

Real Case: Hospital Patient Data Cleaning

The Problem

A hospital wants to analyze patient data but has many issues:

  • 🚫 Missing ages and weights
  • ⚠️ Some patients have age 150 (data entry errors)
  • 📊 Blood pressure readings vary from 60 to 500 (different scales?)
  • 🔴 Some hospital IDs are inconsistent

Cleaning Process

Step 1: Handle Missing Values

# Fill missing age with mean
df['age'].fillna(df['age'].mean())
# Fill missing weight with median
df['weight'].fillna(df['weight'].median())

Step 2: Remove Outliers

# Remove age > 120 (unrealistic)
df = df[df['age'] <= 120]
# Remove BP outliers using IQR
Q3 = df['BP'].quantile(0.75)
df = df[df['BP'] <= Q3 + 1.5 * IQR]

Step 3: Normalize Numeric Data

from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler()
numeric_cols = ['age', 'weight', 'BP']
df[numeric_cols] = scaler.fit_transform(df[numeric_cols])

Step 4: Fix Data Types

# Ensure age is integer
df['age'] = df['age'].astype('int')

✅ Result:

  • • All missing values filled appropriately
  • • Impossible values removed (age > 120)
  • • All numeric data on 0-1 scale (comparable)
  • • Data ready for analysis or modeling!

Complete Data Cleaning Pipeline

  1. Load Data - Import CSV, check shape and types
  2. Check for Issues - Missing values, duplicates, data types
  3. Handle Missing Data - Delete or fill missing values
  4. Fix Data Types - Convert to correct types (int, float, string)
  5. Remove Duplicates - Delete duplicate rows
  6. Handle Outliers - Remove, cap, or transform extreme values
  7. Normalize/Transform - Scale data to proper ranges
  8. Verify Quality - Check descriptive stats, distributions
  9. Save Clean Data - Export for analysis

Activity 2: Detect and Remove Outliers

Find and handle extreme values

📋 Task:

  1. 1. Create dataset with outliers
  2. 2. Calculate Q1, Q3, IQR
  3. 3. Identify outliers using IQR method
  4. 4. Remove outliers
  5. 5. Compare before/after statistics
import pandas as pd
import numpy as np
# Product prices (mostly 50-200, one extreme)
prices = [45, 50, 55, 60, 65, 70, 500] # 500 is outlier!
df = pd.DataFrame({'Price': prices})
# IQR Method
Q1 = df['Price'].quantile(0.25)
Q3 = df['Price'].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
print(f"Lower bound: {lower}, Upper bound: {upper}")
# Find outliers
outliers = df[(df['Price'] < lower) | (df['Price'] > upper)]
print("Outliers:", outliers)
# Remove outliers
df_clean = df[(df['Price'] >= lower) & (df['Price'] <= upper)]
print("Before:", df.describe())
print("After:", df_clean.describe())

✓ Expected Result:

Outlier 500 detected and removed. Mean changes from ~127 to ~62. Data is now consistent!

Activity 4: Complete Data Cleaning Pipeline

Apply all techniques to messy data

📋 Task:

  1. 1. Create realistic messy dataset
  2. 2. Handle missing values
  3. 3. Remove outliers
  4. 4. Normalize numeric columns
  5. 5. Show before/after quality metrics
import pandas as pd
import numpy as np
from sklearn.preprocessing import MinMaxScaler
# Messy e-commerce data
data = {
'Customer': ['A', 'B', 'C', 'D', 'E'],
'Age': [25, np.nan, 35, 150, 40],
'Purchase': [50, 100, 75, 9999, np.nan]
}
df = pd.DataFrame(data)
# BEFORE metrics
print("BEFORE CLEANING:")
print("Missing:", df.isnull().sum())
print(df.describe())
# Step 1: Handle missing
df['Age'] = df['Age'].fillna(df['Age'].mean())
df = df.dropna(subset=['Purchase'])
# Step 2: Remove outliers (Age > 100)
df = df[df['Age'] <= 100]
# Step 3: Normalize
scaler = MinMaxScaler()
df[['Age', 'Purchase']] = scaler.fit_transform(df[['Age', 'Purchase']])
print("AFTER CLEANING:")
print("Missing:", df.isnull().sum())
print(df.describe())

✓ Summary:

  • ✓ 2 missing values handled
  • ✓ 2 rows removed (age 150, purchase 9999)
  • ✓ All numeric data normalized 0-1
  • ✓ Data quality improved!
