Topic 1: Types of Data in Data Mining

🎯 Easy Explanation: What's the Difference?

Think of data like ingredients in a kitchen. Some ingredients are easy to measure (structured), some are messy (unstructured), and some are in between (semi-structured). Data mining requires different approaches for each type.

The Three Types:

🗂️ Structured: Perfectly organized in tables (like Excel spreadsheets)

📄 Semi-structured: Has some organization but not perfect (like JSON files)

📚 Unstructured: No format - just raw content (like videos, images, text)

🗂️

Structured Data

Data organized in rows and columns, with a clear structure. Each field has a defined type and meaning.

💾 Characteristics:

  • • Stored in databases/tables
  • • Easy to search & analyze
  • • Consistent format
  • • ~20% of all data

📊 Examples:

  • • Customer database
  • • Sales records
  • • Employee info
  • • Financial transactions

Example Table:

ID Age City
001 25 Manila
📄

Semi-Structured Data

Data with some organizational structure (like tags or labels), but not as rigid as structured data.

💾 Characteristics:

  • • Has metadata/tags
  • • Variable structure
  • • Flexible format
  • • ~10% of all data

📊 Examples:

  • • JSON files
  • • XML documents
  • • Emails
  • • Web pages

Example JSON:

{
  "user": "John",
  "age": 25,
  "city": "Manila"
}
📚

Unstructured Data

Raw data with no predefined structure. Requires specialized processing to extract meaning.

💾 Characteristics:

  • • No fixed format
  • • Hard to analyze
  • • Needs preprocessing
  • • ~70% of all data

📊 Examples:

  • • Videos & images
  • • Social media posts
  • • Customer reviews
  • • Audio recordings

Processing Challenge:

Requires NLP, Computer Vision, ML to extract useful info

Aspect Structured Semi-Structured Unstructured
Format Tables/Databases JSON, XML Anything goes
Storage Size 20% of all data 10% of all data 70% of all data
Easy to Analyze? ✅ Yes ⚠️ Moderate ❌ Difficult
Analysis Tools SQL, Python Parsers, APIs ML, NLP, CV

Topic 2: CRISP-DM Data Mining Process

🎯 Easy Explanation: What is CRISP-DM?

CRISP-DM stands for Cross-Industry Standard Process for Data Mining. It's like a recipe for baking a cake—a step-by-step guide that ensures you don't miss anything important.

Why use CRISP-DM?

  • ✅ Standardized approach used worldwide
  • ✅ Reduces project risks
  • ✅ Ensures nothing is forgotten
  • ✅ Works for any industry

📊 The 6 Phases of CRISP-DM

1️⃣ Business Understanding

Understand what the business needs and define objectives

2️⃣ Data Understanding

Collect, explore, and understand the available data

3️⃣ Data Preparation

Clean, transform, and prepare data for modeling

4️⃣ Modeling

Apply algorithms to build predictive models

5️⃣ Evaluation

Test models and assess their performance

6️⃣ Deployment

Deploy models to production for real-world use

1

Business Understanding

Establish project objectives and requirements

📋 Tasks:

  • • Meet with business stakeholders
  • • Define problem and success criteria
  • • Identify resources and constraints
  • • Create initial project plan

Example Questions:

• What business problem are we solving?

• How will success be measured?

• What data is available?

2

Data Understanding

Explore and analyze the data

📋 Tasks:

  • • Collect relevant data
  • • Describe data characteristics
  • • Explore data distributions
  • • Identify data quality issues

Example Analysis:

• How many records? (row count)

• What fields exist? (column analysis)

• Any missing values? (data quality)

3

Data Preparation

Get data ready for modeling

📋 Tasks:

  • • Clean data (remove errors)
  • • Handle missing values
  • • Transform variables
  • • Create new features
  • • Select relevant variables

Example Tasks:

• Fill missing age values with median

• Remove duplicate customer records

• Convert text to numbers

4

Modeling

Build machine learning models

📋 Tasks:

  • • Select modeling techniques
  • • Design test strategy
  • • Build model(s)
  • • Assess model parameters

Example Models:

• Decision Tree for classification

• Neural Network for predictions

• K-Means for clustering

5

Evaluation

Assess and validate model performance

📋 Tasks:

  • • Evaluate model results
  • • Compare model performance
  • • Review process
  • • Determine next steps

Evaluation Metrics:

• Accuracy: Does it predict correctly?

• Precision: How many predictions are true?

• ROI: Business value generated?

6

Deployment

Put models into production

📋 Tasks:

  • • Plan deployment strategy
  • • Create monitoring plan
  • • Produce final report
  • • Review project

Example Deployment:

• Launch fraud detection system

• Monitor performance continuously

• Retrain model with new data

🔄 Important: CRISP-DM is Iterative!

CRISP-DM is not a straight line. After deployment, you often loop back to earlier phases to refine the model or improve the solution. This continuous cycle ensures ongoing improvement and adaptation to changing business needs.

Topic 3: Data Mining in Information Systems

🎯 Easy Explanation: Where Does Data Mining Fit?

Information Systems (IS) are all the computer systems, databases, and networks that organizations use to manage data and make decisions. Data Mining is a key part of IS—it's the tool that turns raw data into valuable insights.

Think of IS as a House:

  • 🏠 Database: The foundation (stores data)
  • 🔌 Data Warehouse: The main room (organizes data)
  • 🔍 Data Mining: The detective (finds patterns)
  • 📊 Business Intelligence: The decision maker (uses insights)

🏗️ Information Systems Architecture with Data Mining

Level 1: Data Sources

Raw data from various systems (CRM, ERP, sensors, web logs, etc.)

Level 2: Data Collection & Storage

Databases and Data Warehouses collect and organize data

Level 3: Data Mining & Analysis

Apply techniques to discover patterns and create models

Level 4: Business Intelligence

Create dashboards, reports, and decision support systems

Level 5: Decision Making & Action

Use insights to make business decisions and drive action

🎯 Key Roles of Data Mining in IS

  • 1.

    Extracting Hidden Value

    Uncover patterns humans might miss

  • 2.

    Supporting Decisions

    Provide data-driven insights for better choices

  • 3.

    Predictive Capabilities

    Forecast future trends and behaviors

  • 4.

    Competitive Advantage

    Enable faster, smarter business operations

💼 Business Examples

  • 📱

    Telecom

    Predict which customers will switch providers

  • 🏥

    Healthcare

    Identify high-risk patients for prevention

  • 🛍️

    Retail

    Personalize recommendations for each customer

  • 🏦

    Banking

    Detect fraudulent transactions in real-time

Real-World Case: Spotify's Music Data Mining

The Challenge

Spotify has 500+ million songs and 500+ million users. How do they recommend the perfect song to each person? That's where data mining comes in.

📊 Types of Data Spotify Mines

🗂️ Structured

User account info, play counts, ratings, user demographics

📄 Semi-Structured

Playlist metadata, artist profiles, genre tags, timestamps

📚 Unstructured

User reviews, song lyrics, podcast content, social media comments

🔄 CRISP-DM Applied to Spotify

1. Business Understanding

Goal: Increase user engagement by recommending songs they'll love

2. Data Understanding

Collect 100+ features: listening habits, mood, time of day, genre preferences

3. Data Preparation

Clean data: remove inactive users, handle missing values, normalize features

4. Modeling

Build collaborative filtering model: "Users like you enjoyed these songs"

5. Evaluation

Test: 90% accuracy in predicting which songs users will like

6. Deployment

Launch "Discover Weekly" personalized playlists to millions

💡 Key Result:

Spotify users who follow recommendations listen to an average of 20+ hours per month (vs. 5 hours for non-followers). This increases subscription retention and revenue!

🧪 Hands-On Activity

Step 1: Open Google Colab & Create Dataset

import pandas as pd

import numpy as np

# Create sample dataset

data = {'Student': ['Alice', 'Bob', 'Charlie', 'Diana'],

'Math': [85, 90, 78, 92],

'English': [88, 85, 90, 87],

'Science': [92, 88, 85, 95]}

df = pd.DataFrame(data)

print(df)

Step 2: Basic Data Exploration

# Check data shape

print(f"Shape: {df.shape}")

# Get data info

print(df.info())

# Display statistics

print(df.describe())

# Check first rows

print(df.head())

Step 3: Data Analysis

# Find average scores

print(df[['Math', 'English', 'Science']].mean())

# Find top performer

avg_score = df[['Math', 'English', 'Science']].mean(axis=1)

top_student = df.loc[avg_score.idxmax(), 'Student']

print(f"Top Student: {top_student}")
