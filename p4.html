Topic 1: Descriptive Statistics

🎯 What are Descriptive Statistics?

Descriptive statistics are numbers that describe your data. They summarize large amounts of information into simple, understandable numbers.

Real-world example:

📊 Instead of listing all 500 student test scores, you say: "Average is 75, highest is 98, lowest is 45"

That's much easier to understand!

📊 Key Descriptive Statistics

📈 Mean (Average) Most Used

Add all values and divide by how many values you have

Mean = (Sum of all values) ÷ (Number of values)

Example: Scores: 80, 85, 90, 95

scores = [80, 85, 90, 95]
mean = sum(scores) / len(scores)
print(mean) # 87.5

💡 When to use:

When data is normally distributed (no extreme outliers)

📊 Median (Middle Value) Robust

The middle value when data is sorted (50% above, 50% below)

Example: Sorted scores: 80, 85, 90, 95, 100

import pandas as pd
scores = [80, 85, 90, 95, 100]
median = pd.Series(scores).median()
print(median) # 90

💡 When to use:

When data has outliers or is skewed

🔄 Mode (Most Frequent) Categorical

The value that appears most often in the dataset

Example: Grades: A, B, B, C, B, D → Mode is B (appears 3 times)

from statistics import mode
grades = ['A', 'B', 'B', 'C', 'B']
most_common = mode(grades)
print(most_common) # 'B'

💡 When to use:

With categorical data (colors, grades, categories)

📏 Range (Spread) Simple

Difference between highest and lowest values

Range = Max Value - Min Value

Example: Scores: 45, 68, 75, 90, 98

scores = [45, 68, 75, 90, 98]
range_val = max(scores) - min(scores)
print(range_val) # 53

💡 When to use:

To quickly see how spread out data is

📊 Standard Deviation (Variability) Advanced

Measures how spread out data is from the average. Higher = more spread

Example:

import numpy as np
scores = np.array([80, 85, 90, 95])
std_dev = np.std(scores)
print(std_dev) # ~6.45

💡 Interpretation:

Low std dev = data is consistent | High std dev = data is scattered

⚡ Quick Descriptive Statistics with Pandas

The describe() method gives you all statistics at once:

import pandas as pd
# Create a simple dataset
data = {

}
df = pd.DataFrame(data)

📊 What each line means:

  • count: Number of values
  • mean: Average
  • std: Standard deviation
  • min/max: Lowest/Highest
  • 25%, 50%, 75%: Quartiles (break data into 4 equal parts)

Topic 2: Data Distributions

🎯 What is a Data Distribution?

A distribution shows how data is spread out. It shows if data is clustered at one end, spread evenly, or has a certain pattern.

Think of it this way:

🏪 If you plot how many customers visit a store each hour, the distribution might show:

• Most customers between 10am-2pm (peak hours)

• Very few customers at 8am or 8pm

That's the distribution pattern

⭕ Normal Distribution (Bell Curve)

Most data in nature follows a normal distribution - peak in middle, symmetric on both sides.

📊

Bell Curve - symmetric and centered

Examples of normal distributions:

  • 📏 Height of people in a population
  • 📝 Test scores in a classroom
  • ⏱️ Processing times for transactions
  • 📊 Average daily sales
import numpy as np
import pandas as pd
# Generate normal distribution data
normal_data = np.random.normal(mean=100, std=15, size=1000)
df = pd.DataFrame({'values': normal_data})
print(df.describe())

🔀 Skewed Distributions (Asymmetric)

Sometimes data leans to one side - not symmetric like normal distribution.

➡️ Right Skewed (Positive)

Long tail on the right, peak on the left

📈

Real examples:
Income (few rich), House prices, Web traffic

⬅️ Left Skewed (Negative)

Long tail on the left, peak on the right

📉

Real examples:
Test scores (mostly high), Customer satisfaction, Product quality

import pandas as pd
# Check skewness
skewness = df['values'].skew()
print(f"Skewness: {skewness}")
# Interpretation:
# > 0 = Right skewed
# < 0 = Left skewed
# = 0 = Normal distribution

🔺🔺 Bimodal Distribution

Two peaks - data clusters at two different values

🏔️🏔️

Two peaks instead of one

Real-world example: Student test scores might be bimodal if half studied and half didn't:

  • Peak 1: Low scores (around 50) - students who didn't study
  • Peak 2: High scores (around 85) - students who studied

Topic 3: Patterns and Anomalies

🎯 What are Patterns and Anomalies?

Patterns are trends or behaviors you expect. Anomalies are unexpected things that don't fit the pattern.

Example:

📊 Coffee shop sales: Pattern = high sales during morning (6-9am), low at night

🚨 Anomaly = 3am spike in sales (unusual! Something might be wrong with data or unusual event)

🔍 Finding Patterns in Data

Common patterns to look for:

📈 Trend Pattern

Data consistently increases or decreases over time

Example: Monthly sales increase from Jan → Dec

🔄 Seasonal Pattern

Data repeats at regular intervals

Example: Ice cream sales peak in summer, drop in winter

🔗 Correlation Pattern

Two variables move together (one increases, other increases too)

Example: Study hours and test scores (more study → higher scores)

import pandas as pd
# Find correlation between two columns
correlation = df['study_hours'].corr(df['test_score'])
print(correlation)
# Values close to 1 = strong positive correlation
# Values close to -1 = strong negative correlation
# Values close to 0 = no correlation

🚨 Identifying Anomalies (Outliers)

Anomalies are data points that are very different from the rest.

⚠️ Why Find Anomalies?

  • 🐛 Could be data entry errors
  • 🚨 Could indicate fraud or unusual activity
  • 🔬 Could be interesting discoveries worth investigating

📊 IQR Method (Interquartile Range)

Most common way to find outliers:

Outlier = Any value > (Q3 + 1.5×IQR) OR < (Q1 - 1.5×IQR)

Where:

  • Q1 = 25th percentile (lower quarter)
  • Q3 = 75th percentile (upper quarter)
  • IQR = Q3 - Q1 (middle 50% of data)
import pandas as pd
# Find outliers using IQR
Q1 = df['Score'].quantile(0.25)
Q3 = df['Score'].quantile(0.75)
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR
# Find outliers
outliers = df[(df['Score'] < lower_bound) |
(df['Score'] > upper_bound)]
print("Outliers:", outliers)

📐 Z-Score Method

Measures how many standard deviations away from the mean a value is

Z-Score = (Value - Mean) ÷ Standard Deviation
|Z-Score| > 3 = likely outlier
from scipy import stats
# Calculate Z-scores
z_scores = np.abs(stats.zscore(df['Score']))
outliers = df[z_scores > 3]
print("Outliers:", outliers)

💡 Real Anomaly Example

Student test scores: 72, 75, 78, 80, 82, 85, 87, 90, 92, 5

🚨 Anomaly Found: Score of 5

This is way too low compared to others (even though normal range is 0-100)

What might cause this?

  • ❌ Data entry error (100 mistyped as 5)
  • 🚑 Student was sick during test
  • 🎓 Student didn't show up (entered as failed: 5)
  • ⚠️ System error during grading

Real Case: E-Commerce Sales Analysis

The Business Problem

An e-commerce company has daily sales data for 1 year. They want to understand:

  • 📊 What's typical daily sales?
  • 📈 Is sales trending up or down?
  • 🚨 Are there any unusual days (anomalies)?
  • 🔄 Is there a seasonal pattern?

Data Analysis Steps

Step 1: Descriptive Statistics

df.describe()
# Results: Mean = $5000, Std Dev = $800
# Most days sales are between $4000-$6000

Step 2: Check Distribution

skewness = df['sales'].skew()
# Result: Skewness = 0.3 (slightly right skewed)
# Means a few days with very high sales

Step 3: Find Anomalies

Q1 = df['sales'].quantile(0.25)
Q3 = df['sales'].quantile(0.75)
outliers = df[(df['sales'] > Q3 + 1.5*IQR)]
# Found 12 anomalies: Black Friday, holiday sales, etc

💡 Business Insights:

  • ✅ Normal daily sales average $5000 ± $800
  • ✅ 12 anomalous high-sales days identified (sales events)
  • ✅ Recommend planning inventory based on $5000 baseline
  • ✅ Monitor anomalies to predict special events

import pandas as pd
# Website response times (mostly 200-500ms, one very slow)
response_times = [210, 250, 280, 300, 320, 5000]
df = pd.DataFrame({'Response_ms': response_times})
# IQR Method
Q1 = df['Response_ms'].quantile(0.25)
Q3 = df['Response_ms'].quantile(0.75)

lower_bound = Q1 - 1.5 * IQR

# Find anomalies

(df['Response_ms'] > upper_bound)]

import pandas as pd

IQR = Q3 - Q1
upper_bound = Q3 + 1.5 * IQR
anomalies = df[df['Spending'] > upper_bound]
print("Anomalies found:", anomalies)

✓ Summary of Findings:

  • • High skewness: One customer spent 500 (anomaly)
  • • No missing values
