Algorithm 1 of 3

Decision Trees 🌳

A tree-like model that makes decisions by asking yes/no questions at each node

How Decision Trees Work

Decision trees make predictions by asking a series of yes/no questions about the data, creating a tree-like structure of decisions.

Imagine a teacher deciding if a student will pass: "Did they attend class regularly?" → If yes, "Was their average above 70?" → "Prediction: PASS"

📊 Real Decision Tree Example: Will it Rained Tomorrow?

Is humidity > 80%?
├─ YES: Check wind speed
│ ├─ Wind > 20 km/h? → RAIN (high)
│ └─ Wind ≤ 20 km/h? → POSSIBLE (medium)
└─ NO: Check pressure
├─ Pressure < 1000 mb? → RAIN (high)
└─ Pressure ≥ 1000 mb? → NO RAIN (low)

✓ Strengths

  • • Easy to understand: Visual and interpretable
  • • No preprocessing needed: Works with raw data
  • • Fast predictions: Quick decision making
  • • Handles both: Categorical and numerical data

✗ Weaknesses

  • • Overfitting: Can memorize training data
  • • Instability: Small changes = big differences
  • • Biased data: Performs poorly with imbalanced classes
  • • Greedy algorithm: Doesn't always find best splits

🔑 Key Concepts

Root Node

The first question asked (top of tree)

Internal Nodes

Questions that split the data based on one feature

Leaf Nodes

Final predictions (PASS, FAIL, RAIN, NO RAIN, etc.)

Depth

How many levels deep the tree is (larger depth = more complex = higher overfitting risk)

Algorithm 2 of 3

k-Nearest Neighbors (KNN) 👥

Predicts by finding the k most similar historical examples and using their categories

How KNN Works

KNN finds the k closest neighbors in your training data and makes a prediction based on their category.

Like asking friends with similar taste in music: "If 5 friends like this new song and 1 doesn't, you're more likely to enjoy it!"

📍 Real Example: Movie Recommendations (k=5)

Problem: Will user like movie "Inception"?

Similar User 1: Loved it ⭐
Similar User 2: Loved it ⭐
Similar User 3: Loved it ⭐
Similar User 4: Liked it 👍
Similar User 5: Liked it 👍

Prediction:

5 most similar users rated it positively → User will likely LOVE this movie

✓ Strengths

  • • Flexible: Works with any type of data
  • • No training needed: Learns instantly
  • • Interpretable: Easy to explain decisions
  • • Multi-class: Handles many categories

✗ Weaknesses

  • • Slow predictions: Needs to check all training data
  • • Memory heavy: Must store entire dataset
  • • Curse of dimensionality: Fails with many features
  • • Sensitive to k: Wrong k = poor performance

📊 Choosing k Value

k = 1

Closest single neighbor decides. Simple but noisy.

k = 5 (Good)

Balance between noise and bias. Most common choice.

k = 20+

Smoother but might ignore important patterns.

💡 Rule of Thumb for choosing k

Generally start with k = √n where n = number of training samples

Example: 100 training samples → try k ≈ 10

Algorithm 3 of 3

Naïve Bayes 🎰

A probability-based algorithm that calculates likelihood of each category

How Naïve Bayes Works

Naïve Bayes uses probability to calculate the likelihood that data belongs to each category.

"If 80% of spam emails contain 'BUY NOW', and this email has that phrase, probability it's spam = higher"

🧮 The Basic Idea (Simplified)

Training: Learn what features are common in each category

• Spam emails often have: URGENT, FREE, ACT NOW

• Legitimate emails often have: schedule, meeting, project

Prediction: Calculate probability for each category

• If new email contains URGENT + FREE → Likely SPAM (80% probability)

• If new email contains schedule + meeting → Likely LEGITIMATE (95% probability)

📧 Real Example: Spam Detection

Training Data Analysis:

Spam Emails

  • • 90% contain: "CLICK HERE"
  • • 85% contain: "LIMITED TIME"
  • • 80% contain: "FREE"

Legitimate Emails

  • • 2% contain: "CLICK HERE"
  • • 1% contain: "LIMITED TIME"
  • • 0.5% contain: "FREE"

New Email Analysis:

Contains: "CLICK HERE" + "LIMITED TIME" + "FREE"

Naïve Bayes calculates: Probability of SPAM = 90% × 85% × 80% = VERY HIGH

Prediction: SPAM ✓

✓ Strengths

  • • Very fast: Minimal computation
  • • Works with little data: Even 50-100 samples
  • • Interpretable: Clear probability logic
  • • Good baseline: Reference for other models

✗ Weaknesses

  • • "Naive" assumption: Ignores feature relationships
  • • Zero probability problem: Missing features = 0% probability
  • • Accuracy limits: Often worse than other algorithms
  • • Not for complex patterns: Linear relationships only

🤔 What's "Naïve" About Naïve Bayes?

The algorithm is "naive" because it assumes all features are independent (don't affect each other).

Reality: Features often DO depend on each other. But the algorithm ignores this → called "naive"

Example: In emails, having "CLICK" and "HERE" together isn't random, but Naïve Bayes treats them independently

Evaluating Models

Performance Metrics 📊

Understanding accuracy, precision, recall, and confusion matrices

The Confusion Matrix

A confusion matrixA table showing all four possible prediction outcomes: True Positives, True Negatives, False Positives, False Negatives shows exactly what your model got right and wrong.

Example: Spam Detection (Binary Classification)

Predicted: SPAM Predicted: LEGIT
Actually: SPAM TP: 95 FN: 5
Actually: LEGIT FP: 8 TN: 92

✓ TP (True Positive): Correctly identified SPAM = 95

✗ FP (False Positive): Incorrectly marked LEGIT as SPAM = 8

✗ FN (False Negative): Missed SPAM = 5

✓ TN (True Negative): Correctly identified LEGIT = 92

📈 Accuracy

Accuracy = (TP + TN) / (TP + FP + FN + TN)

What it means: Percentage of all predictions that were correct

Example: (95 + 92) / (95 + 8 + 5 + 92) = 187/200 = 93.5%

✓ Best for: Balanced datasets where all errors matter equally

✗ Worst for: Imbalanced data (e.g., 99% negative class)

🎯 Precision

Precision = TP / (TP + FP)

What it means: Of all predictions we made as SPAM, how many were actually SPAM?

Example: 95 / (95 + 8) = 95/103 = 92.2%

✓ Best for: Spam detection (avoid false alarms), medical false positives

✗ High precision = few false positives but might miss real positives

🔍 Recall

Recall = TP / (TP + FN)

What it means: Of all actual SPAM emails, how many did we catch?

Example: 95 / (95 + 5) = 95/100 = 95%

✓ Best for: Disease detection (don't miss sick patients), fraud detection

✗ High recall = catch all positives but might have false alarms

⚖️ F1-Score

F1-Score = 2 × (Precision × Recall) / (Precision + Recall)

What it means: Balanced average of precision and recall

Example: 2 × (0.922 × 0.95) / (0.922 + 0.95) = 0.936

✓ Best for: When you need both low false positives AND false negatives

✗ Worst for: When one metric is more important than the other

🎓 When to Use Which Metric?

🚨 Fraud Detection:

Use Recall - Catch ALL fraud (better to check some legit transactions than miss fraud)

📧 Spam Filter:

Use Precision - Avoid marking legit emails as spam (better to let spam through than block emails)

🏥 Disease Diagnosis:

Use Recall - Catch ALL sick patients (better false positive than miss disease)

📊 Balanced Data:

Use F1-Score - Good overall performance needed

Part 1

What is Classification?

A machine learning task that predicts categories or classes based on input data.

🔮 Classification Explained

Classification is the process of predicting which category or class a new piece of data belongs to, based on patterns learned from historical data.

Think of it like a teacher grading a test: after seeing many student answers, they learn patterns that help them predict if a new student will get an A, B, C, etc.

📧

Spam Detection

Categories: Spam or Not Spam

Email system classifies incoming emails based on sender, content, keywords

🏥

Disease Diagnosis

Categories: Disease Present or Absent

Medical model classifies patients based on symptoms and test results

🎬

Movie Ratings

Categories: 1-5 Stars or Thumbs Up/Down

Netflix predicts if you'll like a movie based on your history

🔑 Core Concepts

📊 Features

The input variablesCharacteristics or attributes of the data used to make predictions (e.g., email length, sender address, subject keywords) used to make predictions

Example: In spam detection, features include: sender email, word count, presence of links, capital letters percentage

🏷️ Labels (Target)

The category we want to predictThe output or dependent variable that the model tries to predict (e.g., "Spam" or "Not Spam")

Example: For spam detection, the label is either "Spam" or "Not Spam"

The algorithm that learns patterns from training data and makes predictions

Example: Decision Trees, Random Forests, Logistic Regression, Neural Networks

How often the model makes correct predictions

Example: "Model is 95% accurate = 95 out of 100 predictions are correct"

Part 2

Training vs Testing Data

The critical split that determines if your model really works or just memorized answers.

⚠️ The Problem: Overfitting

Imagine a student who memorizes test answers but doesn't understand the material. When they see a slightly different question, they fail.

This is what happens when we train and test on the same data!

The model gets artificially inflated accuracyAccuracy measured on training data is often much higher than real-world performance because the model has "seen" these examples before because it's testing itself on the same data it learned from.

✓ The Solution: Data Splitting

Divide your data into separate training and testing sets so the model learns from one and is evaluated on another.

Common 70/30 or 80/20 split:

70% Training Data

Model learns here

|

30% Testing Data

Model evaluated here

🎓 Training Data

Used to teach the model

  • • Model sees the features AND labels
  • • Learns patterns and relationships
  • • Adjusts its internal parameters
  • • Like a textbook for learning

✅ Testing Data

Used to evaluate the model

  • • Model sees features, predicts labels
  • • True labels hidden until evaluation
  • • Tests if model generalizes
  • • Like a final exam

💼 Real Example: Email Spam Detector

Training Phase (1000 emails):

You show the model 700 emails (some spam, some legitimate). For each email, you provide the correct answer: "This is spam" or "This is not spam." The model learns patterns like common spam keywords, sender info, etc.

Testing Phase (300 emails):

You show the model 300 NEW emails it has never seen. The model makes predictions but doesn't know if they're right. You compare its predictions to the true answers and calculate accuracy.

Result:

If training accuracy = 99% but testing accuracy = 78%, the model is overfitting. It memorized training data but can't predict new emails well.

🔄 Advanced: K-Fold Cross-Validation

A more robust method: split data into 5 or 10 parts, train and test multiple times:

Fold 1: Train on 4/5, Test on 1/5
→
Fold 2: Train on 4/5, Test on 1/5
→
Fold 3: Train on 4/5, Test on 1/5
...

Average all results for more reliable accuracy estimate

Step-by-step process from raw data to predictions.

🗂️ Data Collection

2

🧹 Data Preprocessing

Clean and prepare data for modeling

  • • Remove duplicates and missing values
  • • Convert text to numbers (encoding)
  • • Scale features to similar ranges
3

✂️ Train/Test Split

Divide data into training and testing sets

4

🤖 Model Selection & Training

Choose an algorithm and train it on training data

  • • Choose algorithm (Decision Tree, Random Forest, etc.)
  • • Feed training data to algorithm
  • • Model learns patterns from 7,000 emails
5

📊 Model Evaluation

Test model on unseen data and measure performance

  • • Use 3,000 test emails (model has never seen these)
  • • Model makes predictions
  • • Compare predictions to true labels
  • • Calculate accuracy, precision, recall, F1-score
6

🔧 Hyperparameter Tuning

Adjust settings to improve performance (optional)

If accuracy is low (e.g., 75%), try different algorithm parameters and retrain. Repeat until satisfied.

7

🚀 Deployment

Use trained model to make predictions on new, unseen data

Part 4

Popular Classification Algorithms

Different approaches to solve classification problems.

🌳

Decision Trees

Makes decisions by asking a series of yes/no questions

How it works: "Is word count > 100?" → "Does it have links?" → "Prediction: Spam"

✓ Pros:

  • • Easy to understand
  • • Fast prediction

✗ Cons:

  • • Can overfit easily
  • • Unstable
🌲

Random Forest

Combines many decision trees for better predictions

How it works: Creates 100 decision trees, each votes, final prediction = majority vote

✓ Pros:

  • • Very accurate
  • • Resists overfitting

✗ Cons:

  • • Slower to predict
  • • Hard to interpret
📈

Logistic Regression

Uses a mathematical formula to predict probabilities

How it works: Calculates probability of being spam: 0.0 (definitely not) to 1.0 (definitely spam)

✓ Pros:

  • • Fast & simple
  • • Interpretable

✗ Cons:

  • • Linear relationships only
  • • Limited accuracy
👥

K-Nearest Neighbors (KNN)

Predicts by finding similar historical examples

How it works: Finds 5 most similar emails, predicts based on their categories

✓ Pros:

  • • Simple concept
  • • Flexible

✗ Cons:

  • • Slow predictions
  • • Needs lots of memory
